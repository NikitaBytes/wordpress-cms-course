## 🎯 Цель и задачи работы

Целью данной лабораторной работы было научиться создавать собственную тему WordPress, разобраться в её минимальной структуре и понять принципы работы шаблонов в системе WordPress.

**В рамках выполнения работы требовалось:**

- Перейти в папку тем WordPress и создать собственную директорию темы
- Включить режим отладки в WordPress (`WP_DEBUG`)
- Создать обязательные файлы темы (`style.css` и `index.php`)
- Реализовать основные шаблоны и вынести общие части сайта в отдельные файлы
- Подключить стили темы через `functions.php`
- Создать дополнительные шаблоны для записей, страниц, архива, боковой панели и комментариев
- Стилизовать тему
- Добавить превью темы `screenshot.png`
- Активировать тему в административной панели WordPress и проверить её работу

## 📝 Краткое описание выполненной работы

В ходе выполнения лабораторной работы я использовал уже существующую локальную установку WordPress, развёрнутую в Docker в предыдущей лабораторной работе. Это позволило мне не создавать новую среду с нуля, а сразу перейти к разработке темы внутри уже настроенного WordPress-сайта.

Я создал собственную тему usm-theme в директории wp-content/themes, включил режим отладки WP_DEBUG, добавил обязательные файлы style.css и index.php, после чего реализовал остальные шаблоны: header.php, footer.php, sidebar.php, single.php, page.php, comments.php, archive.php и functions.php. Далее я оформил тему базовыми стилями, добавил файл-превью screenshot.png, активировал тему в админ-панели WordPress и проверил отображение сайта на главной странице, на странице отдельной записи, на обычной странице и в архиве записей.

## 📘 Краткая документация к теме

### Название и назначение

**Название:** USM Theme

**Назначение:** Тема предназначена для учебной лабораторной работы и демонстрирует минимальную структуру пользовательской темы WordPress.

### Реализованные шаблоны

- `index.php` — главная страница и fallback-шаблон
- `single.php` — отдельная запись
- `page.php` — отдельная страница
- `archive.php` — архив записей
- `header.php` — шапка сайта
- `footer.php` — подвал сайта
- `sidebar.php` — боковая панель
- `comments.php` — комментарии
- `functions.php` — подключение стилей

### Особенности реализации

- Стили темы подключаются через `wp_enqueue_style()`
- Общие части шаблона подключаются через стандартные функции WordPress
- На главной странице выводятся 5 последних записей
- В теме присутствует собственное превью `screenshot.png`

## 🚀 Инструкции по запуску проекта

Если WordPress уже развёрнут в Docker, для запуска проекта:

1. Запустить Docker-контейнеры WordPress, MySQL и phpMyAdmin
2. Убедиться, что WordPress доступен по локальному адресу
3. Поместить папку `usm-theme` в директорию `wp-content/themes/`
4. Перейти в административную панель WordPress
5. Открыть раздел **Appearance → Themes**
6. Найти тему **USM Theme** и активировать её

## ❓ Контрольные вопросы

| №   | Вопрос                                           | Ответ                                                                                                                                                                                       |
| --- | ------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Какие два файла обязательны для темы WordPress?  | Для любой темы WordPress обязательны два файла: `style.css` и `index.php`. Файл `style.css` содержит метаданные темы и стили, а `index.php` является главным шаблоном темы.                 |
| 2   | Как подключаются общие части шаблонов?           | Общие части подключаются стандартными функциями WordPress: `get_header()`, `get_footer()` и `get_sidebar()`. Это позволяет переиспользовать части шаблона в разных файлах.                  |
| 3   | Чем отличаются index.php, single.php и page.php? | `index.php` — основной шаблон и fallback. `single.php` — отдельная запись блога. `page.php` — отдельные страницы сайта.                                                                     |
| 4   | Зачем нужен файл functions.php в теме?           | `functions.php` добавляет функциональность темы: подключение стилей/скриптов, регистрация меню, виджеты, поддержка миниатюр. В данной работе использован для подключения файла стилей темы. |
