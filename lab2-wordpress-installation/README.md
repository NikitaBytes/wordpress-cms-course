### 1️⃣ Подготовка локальной среды

На первом этапе я подготовил локальную среду для работы с WordPress. Вместо XAMPP или MAMP я использовал Docker, так как этот вариант также допускается в задании и более удобен для работы на macOS.

Я создал отдельную рабочую директорию для лабораторной работы и подготовил файл docker-compose.yml, в котором были описаны три сервиса:

- контейнер с MySQL для базы данных;
- контейнер с WordPress;
- контейнер с phpMyAdmin для управления базой данных через браузер.

После этого я запустил контейнеры с помощью команды:

```bash
docker compose up -d
```

После успешного запуска я убедился, что все сервисы работают корректно, а контейнеры находятся в состоянии running.

## ❓ Контрольные вопросы

| №     | Вопрос                                                            | Ответ                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ----- | ----------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **1** | **Что делает тема в WordPress, а что — плагин?**                  | Тема в WordPress отвечает за внешний вид сайта: оформление страниц, расположение блоков, стили, шрифты, цвета, шапку и подвал сайта. При смене темы меняется именно визуальное представление сайта. Плагин добавляет в WordPress новую функциональность. Например, плагины могут создавать формы обратной связи, добавлять SEO-возможности, кэширование, безопасность, новые редакторы и другие функции, которых нет в системе по умолчанию. Иначе говоря, тема отвечает в первую очередь за дизайн, а плагин — за дополнительные возможности. |
| **2** | **Почему при смене темы контент сайта не теряется?**              | Контент WordPress хранится в базе данных отдельно от темы. В базе данных сохраняются страницы, записи, комментарии, пользователи и другие данные сайта. Тема только отвечает за то, как этот контент будет отображаться на сайте. Поэтому при смене темы содержимое сайта не удаляется: меняется только способ его визуального представления.                                                                                                                                                                                                  |
| **3** | **Как можно изменить внешний вид сайта без редактирования кода?** | Изменить внешний вид сайта без редактирования кода можно несколькими способами: • установить и активировать другую тему; • использовать раздел Customize для изменения внешнего вида; • изменить логотип, заголовок, цвета и другие параметры через настройки темы; • использовать дополнительные визуальные настройки, встроенные в тему или конструктор страниц. Таким образом, WordPress позволяет вносить многие изменения во внешний вид сайта через административную панель, без ручного редактирования файлов.                          |
